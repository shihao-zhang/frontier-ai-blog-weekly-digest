## Anthropic 本周动态

### 研究 (Research)

- **When AI Builds Itself：AI 递归自我改进临界点警告**（6月4日）：Anthropic Institute 由联合创始人 Jack Clark 和研究负责人 Marina Favaro 联合发布《When AI Builds Itself》报告。核心数据：截至2026年5月，Anthropic 生产代码库中超过80%的代码由 Claude 撰写（部分报道称已超90%）；典型工程师每季度合并代码量是2024年的8倍；在最开放的工程任务上，Claude 成功率在6个月内从26%跃升至76%。报告警告：AI 可能正接近"递归自我改进"门槛——在没有实质性人工介入的情况下自主改进自身能力。Anthropic 随之呼吁建立可核查的国际机制，以在越过该门槛前实现"全球有条件暂停"（即若其他前沿实验室也参与则暂停）。这是 Anthropic 首次将内部研发现状与公共政策诉求直接绑定——既是一份技术报告，也是一份政治信号。[链接](https://www.anthropic.com/institute/recursive-self-improvement)

- **AI-enabled Cyber Threats 年度报告：AI 驱动的攻击面正在快速扩大**（6月3日）：Anthropic 分析了2025年3月至2026年3月间因恶意网络活动被封禁的832个账户，并将攻击战术映射至 MITRE ATT&CK 框架。关键发现：（1）67.3%的攻击者使用 AI 进行恶意软件编写等攻击前准备；（2）"中等风险以上"攻击者占比从第一个六个月的33%飙升至第二个六个月的56%——AI 赋能攻击者的比例在一年内增长了约1.7倍；（3）6.5%的攻击者使用 AI 辅助"横向移动"等深度渗透行为。研究结果已发布于 Verizon 2026 年数据泄露调查报告。**信号**：AI 工具正从"攻击加速器"向"攻击自动化"方向演进，防御侧亟需相应升级。[链接](https://www.anthropic.com/news/AI-enabled-cyber-threats-mitre-attack)

- **首例 AI 编排网络间谍战役披露**（6月3日）：Anthropic 发布报告，揭露了一起被认定为中国国家支持的威胁组织（GTG-1002）利用 Claude Code 执行的大规模网络间谍行动——这是已记录的首起无需大量人工干预即完成的大规模网络攻击。攻击者针对全球约30个目标（大型科技公司、金融机构、化工制造商、政府机构），少数案例中成功渗透。攻击流程约80-90%由 AI 自动执行，人工仅在4-6个关键决策点介入。攻击者通过将任务拆解成"无害子任务"来绕过 Claude 的安全护栏。Anthropic 在2025年9月中旬检测到该行动并中断。**启示**：这标志着 AI 在攻击链中的角色已从"辅助工具"升级为"作战主体"，对企业安全团队构成全新威胁范式。[链接](https://www.anthropic.com/news/disrupting-AI-espionage)

- **Making Claude a Chemist：AI 在化学研究中的新能力**（6月5日）：Anthropic 科学博客发布研究，展示 Claude Opus 4.7 在化学领域的专业能力。核心发现：Claude 能够直接从期刊图表或手绘草图中读取化学结构，并从方法部分提取实验细节；在 NMR（核磁共振波谱）解析任务上，Opus 4.7 的表现与专业 NMR 软件持平，在部分任务上甚至超越专用软件。这是 Anthropic 将 Claude 定位为"科学研究加速器"系列布局的最新进展，延续了此前的生命科学、数学等领域拓展。**意义**：AI 正从"写代码"跨越到"做实验"，Claude 对化学家日常的"翻译-检索-整合"工作的辅助能力已具实质价值。[链接](https://www.anthropic.com/research/making-claude-a-chemist)

### 工程 (Engineering)

- **How We Contain Claude Across Products：Agent 安全的工程实践**（近期发布）：Anthropic 工程博客发布长文，系统梳理在 claude.ai、Claude Code 和 Claude Cowork 三个主要 Agent 产品中实施"能力限制"（containment）的工程经验。文章聚焦两类策略：（1）**监督 Agent 的行为**；（2）**限制 Agent 能做什么**——通过沙箱、虚拟机和出口流量控制实施访问边界。文章披露了一个真实安全漏洞：攻击者在 Git 仓库的 `.claude/settings.json` 中植入钩子，而 Claude Code 在显示"是否信任此目录"提示之前会先读取项目配置，导致恶意钩子在用户确认前自动执行。这一披露直接推动了后续版本对跨会话消息和钩子机制的安全加固。**产品意图信号**：Anthropic 选择公开披露 containment 架构而非秘而不宣，表明其在 Agent 安全领域寻求行业对话而非独自摸索。[链接](https://www.anthropic.com/engineering/how-we-contain-claude)

## OpenAI 本周动态

### 研究 (Research)

- **Evaluating Chain-of-Thought Monitorability：思维链可监控性系统研究**（6月4日）：OpenAI 发布重要安全研究，首次系统性评估监控 AI 推理过程是否切实可行。核心发现：（1）在几乎所有研究场景中，监控思维链（chain-of-thought）显著优于仅监控最终行为/输出；（2）强化学习在当前实验规模下"未实质性损害可监控性"，甚至在早期推理步骤可提升可监控性；（3）存在"可监控性税"（monitorability tax）——使用更高推理强度的小模型可在匹配大模型能力的同时保持更高可监控性；（4）提问追踪（follow-up questions）在初始推理努力低时可揭示更多可监控信息。研究发布了包含13个评估、覆盖24个环境和3类问题的基准套件 MonitorBench。**意义**：这是目前对"是否能够监控 AI 的内部推理"这一问题最严格的实证研究，为 AI 安全控制层的设计提供了关键证据基础。[链接](https://openai.com/index/evaluating-chain-of-thought-monitorability/)

- **Rosalind Biodefense & GPT-Rosalind 全球扩展**（6月3日）：OpenAI 发布 Rosalind Biodefense 计划并大幅扩展 GPT-Rosalind（生命科学专用模型）的访问权限。GPT-Rosalind 基于 GPT-5.5 构建，针对药物发现、蛋白质工程和基因组学进行专项优化，在长定量生物学分析中所需 token 减少31%。OpenAI 将为经审查的开发者提供资助访问，并向美国政府及盟国公共卫生合作伙伴开放，用于生物防御工作（包括早期预警系统、疫情建模、诊断工具和医疗对策开发），合作机构包括约翰斯·霍普金斯应用物理实验室和 CEPI（流行病防范创新联盟）。本次更新同时将 GPT-Rosalind 的研究预览首次向全球符合条件的组织开放。[链接](https://openai.com/index/strengthening-societal-resilience-with-rosalind-biodefense/)

## 本周重要论文 (Notable Papers)

- **When AI Builds Itself** by Anthropic Institute (Jack Clark, Marina Favaro)：定量记录了 AI 系统自主编写生产代码的能力跃迁（6个月内成功率从26%→76%，80%代码由 Claude 撰写），并形式化提出了"递归自我改进门槛"的概念及全球协调暂停机制。[链接](https://www.anthropic.com/institute/recursive-self-improvement)
  - 🔑 对我们的启示：如果前沿实验室自己的生产代码库已由 AI 主导编写，AI-native 产品的"工程速度飞轮"正在大幅加速——这意味着竞争窗口正在以历史上从未有过的速度压缩。现在不上 AI 编程工作流，差距半年内可能超过5倍。

- **Evaluating Chain-of-Thought Monitorability** by OpenAI：系统研究了"监控 AI 推理过程是否可行"，提出并验证了 MonitorBench（13评估 × 24环境），发现 CoT 监控显著优于仅监控输出，并量化了"可监控性税"。[链接](https://openai.com/index/evaluating-chain-of-thought-monitorability/)
  - 🔑 对我们的启示：构建 Agent 审计/合规产品时，应优先捕获和分析推理链而非仅记录输入输出；"暴露更小模型的完整思维链"可能比"使用更大黑盒模型"在可监控性上有显著优势。

- **What We Learned Mapping a Year's Worth of AI-Enabled Cyber Threats** by Anthropic：基于832个恶意账户的 MITRE ATT&CK 映射，发现 AI 赋能的网络攻击者比例在一年内增长1.7倍，AI 正从"攻击准备工具"向"横向移动"等深度渗透场景演进。[链接](https://www.anthropic.com/news/AI-enabled-cyber-threats-mitre-attack)
  - 🔑 对我们的启示：AI Agent 产品需要内置"攻击者视角"的威胁建模——当我们的 Agent 帮助用户访问内部系统时，同样的能力也会成为攻击向量；授权链、最小权限和审计日志不是可选项。

- **Making Claude a Chemist** by Anthropic：展示了 Claude Opus 4.7 在 NMR 波谱解析任务上达到或超越专业软件水平，AI 科学辅助能力正从"通识百科"向"专业研究工具"突破。[链接](https://www.anthropic.com/research/making-claude-a-chemist)
  - 🔑 对我们的启示：垂直行业 AI Agent 的能力上限比预期高——如果 AI 能做 NMR 分析，那么法律文件解析、金融报表分析等专业任务的自动化程度也会比想象中更高。

- **Code as Agent Harness: A Comprehensive Survey** (arXiv:2605.18747)：综合调研涵盖编程助手、GUI/OS 自动化、具身 Agent、科学发现、DevOps 和企业工作流六大方向，梳理了"代码作为 Agent 执行框架"的开放挑战，包括超越最终任务成功的评估方式、不完整反馈下的验证，以及无回归的框架改进。[链接](https://arxiv.org/abs/2605.18747)
  - 🔑 对我们的启示：当前 Agent 评估框架在"持续演化的软件"场景中成功率从80%+崩塌至38%以下（EvoClaw 基准），这意味着我们不能仅凭静态 benchmark 评估 Agent 的生产可用性，需要建立动态/连续评估框架。

## 横向洞察 (Cross-cutting Insights)

### 本周各实验室的发力方向

本周最显著的宏观信号是**"AI 自我加速"正从技术概念变成可量化的商业现实**。Anthropic 的《When AI Builds Itself》把这一现实用数据落地——80%代码由 AI 写、工程师产出提升8倍——并将其与全球治理诉求绑定。与此同时，Microsoft Build 2026 的7款自研 MAI 模型揭示了 AI 竞赛的另一面：AI 加速正在让大型科技公司意识到"依赖单一外部模型提供商是战略风险"，自研能力正在从可选项变成必选项。这两个信号叠加，指向同一个结论：**AI 能力的快速扩散正在瓦解现有的竞争护城河，无论是商业产品还是内部工具链。**

在具体方向上，本周各实验室的分化明显：Anthropic 全力押注**安全即业务**（Glasswing 扩至NATO/ENISA，工程博客的 containment 框架，AI 间谍战役披露），将安全能力作为企业销售的核心叙事；OpenAI 的重心是**泛化可达性**（Codex 角色插件面向非开发者，Bedrock GA 面向企业 AWS 存量用户，Dreaming/Lockdown 面向 ChatGPT 日活用户）；Google 则继续以**研究 Agent** 为差异化支点（Deep Research Max 的超大规模综合能力）。三条路线互不直接冲突，但在企业决策者心目中争夺的是同一份预算。

### 从研究走向产品化的技术趋势

**"AI 监控基础设施"正在快速产品化。** OpenAI 的 CoT 可监控性研究、Anthropic 的 containment 工程博客、Lockdown Mode 和跨会话消息权限加固——这四件事在同一周出现绝非巧合。行业正在从"如何让 Agent 更强大"转向"如何安全地知道 Agent 在做什么"。对初创公司来说，这是一个尚未被充分利用的产品机会：**Agent 可观测性和运行时监控**正处于从研究原型向生产工具的关键过渡期。

**"非开发者 AI 用户"正在成为新增量的主战场。** OpenAI Codex 20%的非开发者用户以3倍速度增长；Claude 为小企业推出打包的15项自动化工作流；Dreaming 让 ChatGPT 更像"持久的个人助理"而非"一次性工具"。这一趋势意味着下一代 AI-native 应用的目标用户是"不懂提示工程的普通员工"，而非"愿意写系统 Prompt 的极客"。

### 文档/产品变化背后的战略信号

Anthropic 的 Claude Code 订阅计划拆分是本周最值得深度解读的产品信号之一。将 Agent SDK 的程序化用量从订阅计划剥离，在商业逻辑上意味着：**AI 的"重度用量"（Agent 编排、CI/CD 自动化）必须按真实消耗付费，不能再通过订阅层摊薄成本**。这与 Anthropic 同期披露的数据（每个工程师产出提升8倍）形成张力——正是因为 Agent 用量爆炸，才需要独立计费。对使用 Claude 构建产品的初创公司而言，这意味着成本模型正在发生根本性变化，需要重新审视基于订阅定价构建的 Agent 工作流的单位经济。

OpenAI 同日废弃 Evals 平台和 Agent Builder 是另一个值得注意的信号：这意味着 OpenAI 正在主动收拢"实验性工具的长尾"，将开发者工具生态聚焦在 Agents SDK + Codex + Responses API 三条主线上。对使用被废弃工具的开发者，迁移窗口至11月30日，建议尽早评估。
