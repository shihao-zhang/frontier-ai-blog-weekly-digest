## Anthropic 本周动态

### 研究 (Research)

- **Paving the way for agents in biology**（6月8日）：Anthropic 发布研究文章，揭示了当前生物 AI Agent 的一个关键瓶颈——不是模型能力，而是数据基础设施。测试显示，包括 Claude Sonnet 4 和 GPT-5.5 在内的前沿模型在病毒序列检索任务（VirBench，120个查询，涉及40种病原体）上的准确率仅为 16.9%–91.3%，且同一模型在重复运行中结果差异悬殊。Anthropic 与 NCBI 合作开发了 gget virus——一个确定性检索工具——将 AI Agent 的准确率提升至 99.7%，并彻底消除了结果不稳定性。Anthropic 将 gget virus 定位为"上下文引擎"（context engines）概念的一个实例：Agent 需要可靠的、专门构建的数据接口，而非通用的网络搜索工具，才能在生命科学领域可靠工作。Biomni（斯坦福大学的 Agent 平台，覆盖25+生物子领域）是同类概念的另一实践。**意义**：这篇文章标志着 Anthropic 正在系统性地为 Claude Agent 在生命科学领域的落地扫清障碍，构建"Agent 就绪的生物数据基础设施"。[链接](https://www.anthropic.com/research/agents-in-biology)

### 工程 (Engineering)

- **Claude for Apple Foundation Models 框架**（6月8日）：Anthropic 发布了一个 Swift 软件包，允许 Apple 开发者将 Claude 集成进 Apple 的 Foundation Models 框架（iOS 27、iPadOS 27、macOS 27、visionOS 27、watchOS 27）。其核心设计思路是分层调用：Apple 的设备端小模型处理轻量推理，复杂任务通过该包无缝切换至 Claude API，开发者无需手动管理切换逻辑。该包处理流式传输、工具调用和结构化响应，并将结果直接回传至 SwiftUI 视图。**产品意图信号**：这是 Anthropic 打入 Apple 生态的最直接动作——随着 Apple Intelligence 平台的扩大，这个 SDK 让 Claude 成为所有 Apple 平台的后端大脑候选。[链接](https://claude.com/blog/claude-for-foundation-models)

### 产品 (Product)

- **Claude Fable 5 + Claude Mythos 5 正式发布**（6月9日）：本周最重大事件。Anthropic 发布了 Claude Fable 5——历史上首个向公众开放的 Mythos 级模型（此前 Mythos Preview 从未公开发布过）。同日，Anthropic 还推出了 Claude Mythos 5：与 Fable 5 使用相同底层模型，但部分安全护栏被放开，仅向小范围网络防御机构和关键基础设施提供商开放。

  Fable 5 的核心能力亮点：（1）**软件工程**：在 SWE-bench 等编程基准上超越所有可用模型；Stripe 用它在一天内完成了一个5000万行代码库的迁移。（2）**长上下文与记忆**：原生支持 100 万 token 上下文，并配备基于文件的持久记忆机制；在纸牌游戏《Slay the Spire》的内部测试中，启用持久记忆后的表现是 Opus 4.8 的3倍。（3）**视觉能力**：在科学图表数字提取和截图重建 Web 应用方面达到 SOTA；仅凭视觉信号完成了《Pokémon FireRed》通关。（4）**安全架构**：配备域名安全分类器，在网络安全、生物、化学、蒸馏等高风险领域自动回退至 Opus 4.8，触发回退的会话比例 <5%。

- **Claude Managed Agents：定时调度 + 环境变量保险库（Vaults）**（6月9日）：Anthropic 为 Claude Managed Agents 推出两项新能力，现已进入公开 Beta：（1）**Scheduled Deployments**——Agent 可按 Cron 表达式自动触发，无需人类在线，每次调度触发一个新的 Agent 会话并完成任务；（2）**Vaults**——安全存储 Agent 所需的环境变量（API 密钥、凭证等），Agent 运行时按需访问，无需将敏感信息硬编码在提示词或代码中。案例：Milana 用 Vaults 让其 AI 产品工程师 Agent 自动连接到客户代码库，实现无人值守的 Bug 查找与修复。**产品意图信号**：定时调度 + Vaults 意味着 Claude Agent 首次具备了企业级的"无人值守运行"能力，正式从"工具"走向"员工"——可以在业务系统中独立持续运转。[链接](https://claude.com/blog/whats-new-in-claude-managed-agents)

## 本周重要论文 (Notable Papers)

- **"Agentic Software: How AI Agents Are Restructuring the Software Paradigm"** by 多位作者 (arXiv:2606.05608, 6月2026)：论文形式化了"传统软件"与"Agent 软件"的本质区分——传统软件中代码是预写决策逻辑的载体；Agent 软件中 Agent 本身是软件，其决策逻辑在运行时动态生成。论文梳理了从授权软件 → SaaS → AaaS（Agent-as-a-Service）的历史弧线，并讨论了这一范式转换对软件工程、架构和商业模式的影响。[链接](https://arxiv.org/abs/2606.05608)
  - 🔑 对我们的启示：AI-native 产品的核心竞争力不在于"写出更好的代码"，而在于"构建更好的 Agent"——Agent 即产品，其决策逻辑的质量取决于训练和系统设计而非源码审查。

- **"Memory OS of AI Agent"** (arXiv:2506.06326, 6月2026)：提出 Agent 记忆的操作系统级抽象，将 Agent 记忆问题分类为四类查询：Single-hop（单步检索）、Multi-hop（多跳推理）、Temporal（时序相关）和 Open-domain（开放域），并构建相应的评估框架。[链接](https://arxiv.org/abs/2506.06326)
  - 🔑 对我们的启示：Agent 的"记忆管理"需要操作系统级的设计思维，不同类型的业务任务对记忆的要求差异显著——数字员工产品的记忆架构应按查询类型分层设计。

- **"A Survey of Evaluation Tools for AI Assistants and Agents"** (arXiv:2506.08800, 6月2026)：系统综述了当前用于评估 AI 助手和 Agent 的工具生态，涵盖编程、推理、多步骤任务等领域的主流评估框架和基准。[链接](https://arxiv.org/abs/2506.08800)
  - 🔑 对我们的启示：Agent 评估工具正在快速成熟，这意味着"Agent 能力测量"将从艺术变为工程——企业在采购/评估 AI Agent 产品时将有更标准化的参照系。

- **"Agentic Software Rethinking Software Engineering for Agentic AI Systems"** (arXiv:2604.10599)：重新审视在 AI Agent 时代的软件工程实践，探讨传统 SE 工具（代码审查、测试、版本控制、架构设计）如何需要演进以适应 Agent 驱动的开发范式。[链接](https://arxiv.org/abs/2604.10599)
  - 🔑 对我们的启示：Agent 时代的工程规范尚未定型，率先建立"Agent 工程最佳实践"的团队将拥有显著的生产效率优势。

## 横向洞察 (Cross-cutting Insights)

### Mythos 级能力公开：能力民主化的加速

本周最标志性的事件是 Claude Fable 5 的发布。这不仅是一个新模型——它代表着此前 Anthropic 认为"太危险而无法公开"的能力级别首次向公众开放，且以内置安全分类器的方式解决了这一矛盾：用域级回退（fallback to Opus 4.8）代替全面封锁，将安全从"能力门槛"变为"能力调节器"。这一模式如果被证明可行，将为整个行业设立一个新参照：前沿能力不必在"完全封锁"与"完全开放"之间二选一，而是可以通过精细化的域名安全架构实现"有条件开放"。与此同时，两家最强的前沿实验室——Anthropic（6月1日）和 OpenAI（6月8日）——在同一周内相继提交 IPO 申请，进入公开资本市场的前夜，这将进一步加速两者之间的能力披露竞争。

### Agent 持久化：本周的隐线主题

将本周 Anthropic 和 OpenAI 的产品动作串联起来，会发现它们指向同一个技术目标：**让 Agent 从"短任务响应者"变为"持续工作者"**。Anthropic 的 Managed Agents 定时调度（Cron + Vaults）、Claude Code 的嵌套 Sub-Agent（5层），与 OpenAI 收购 Ona（让 Codex 在客户云环境中持久运行数小时/数天）——这些动作的底层逻辑完全一致：打破"用户必须在线才能让 Agent 工作"的约束。这是企业级 AI 采购决策中的关键摩擦点，谁先解决，谁就在企业市场获得先机。Anthropic 通过 Managed Agents 平台方式解决，OpenAI 通过收购 Ona 获取基础设施控制权，两条路都会通向同一个目标。

### 从语言到科学：专域模型的崛起
