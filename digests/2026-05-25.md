## OpenAI 本周动态

### 研究 (Research)

- **AI 模型自主推翻 Erdős 80年前几何猜想**（5月20日）：OpenAI 的内部通用推理模型自主推翻了离散几何中的经典核心猜想——Paul Erdős 1946年提出的平面单位距离问题。该问题问：在平面上放置 n 个点，最多有多少对点的距离恰好为1？近80年来数学界认为正方形网格是最优排列。OpenAI 模型利用深层代数数论（Golod-Shafarevich 定理和无穷类域塔）找到了实现多项式级改进的全新无穷构造族，并由 Princeton 数学家 Will Sawin 等外部数学家独立验证。Fields 奖得主 Tim Gowers 称之为"AI 数学的里程碑"。这是 AI 首次自主解决某数学子领域的核心开放问题。[链接](https://openai.com/index/model-disproves-discrete-geometry-conjecture/)

- **推进内容溯源标准**（5月19日）：OpenAI 加入 C2PA 指导委员会，成为 C2PA 合规生成方，对 ChatGPT、Codex 和 API 生成的所有图像添加加密签名元数据；同时与 Google DeepMind 合作，将 SynthID 隐形水印（可抵抗截图和格式转换）集成进所有生成图像；并发布公开预览验证工具，供任何人检测图像来源。这构建了 C2PA 元数据 + SynthID 水印的双层溯源防护体系，是业界迄今最完整的生成内容溯源方案。[链接](https://openai.com/index/advancing-content-provenance/)

## 本周重要论文 (Notable Papers)

- **SkillOpt: Executive Strategy for Self-Evolving Agent Skills** by Microsoft Research（5月22日）：将一个紧凑的 Markdown 文档作为冻结 LLM Agent 的"可训练参数"——不更新模型权重，通过 rollout、反思、有界编辑和留出集验证来优化技能文档。在52个评测格（6个基准 × 7个模型 × 3个执行环境）上全部达到最优或并列最优，使 GPT-5.5 在直接对话中平均精度提升 +23.5 点，在 Codex Agent 循环中提升 +24.8 点，在 Claude Code 中提升 +19.1 点。技能产物可跨模型规模和执行环境无成本迁移。[链接](https://arxiv.org/abs/2605.23904)
  - 🔑 对我们的启示: 这直接实现了"自进化数字员工"范式——Agent 通过经验不断优化自身操作手册，生成的 `best_skill.md` 可直接作为系统提示复用，无需重新训练，是构建持续改进型 AI Worker 的关键技术路径。

- **Automated Benchmark Auditing for AI Agents and LLMs** (ABA)（约5月25日）：对9个领域168个基准进行系统审计，发现超过25.7%的任务存在关键问题（模糊任务设计、执行环境冲突、错误 ground truth）。过滤问题任务后，SWE-bench Verified 和 Terminal-Bench 2 上的模型平均性能分别提升9.9%和9.6%，部分模型排名发生显著变化。[链接](https://arxiv.org/abs/2605.26079)
  - 🔑 对我们的启示: 四分之一的 Agent 基准任务本身有问题，当前排行榜数字系统性偏低——在以基准分数做 Agent 能力决策时需要更高的审慎度，内部评测集的可靠性比外部排名更重要。

- **Constraint Decay: The Fragility of LLM Agents in Backend Code Generation** by Francesco Dente et al.（5月7日发布，本周广泛传播）：发现随着多文件后端代码生成任务中的结构约束累积，Agent 性能显著下降——即"约束衰减"失效模式。揭示了编程 Agent 在复杂、相互依赖代码库中无法维护一致性的具体原因。[链接](https://arxiv.org/abs/2605.06445)
  - 🔑 对我们的启示: 软件工程 Agent 在多文件依赖场景下存在系统性薄弱点，"AI 软件工程师"产品需要专门的约束追踪和一致性验证机制——而不能只依赖更大的上下文窗口。

- **The Evolution of Tool Use in LLM Agents: From Single-Tool Call to Multi-Tool Orchestration**（3–4月发布，本周广泛引用）：综述从单工具调用到非线性多工具依赖图的演进，指出传统自回归生成在面对数千个 API 和级联错误时的根本局限，为 Agent 推理范式的重新设计提供论证框架。[链接](https://arxiv.org/pdf/2603.22862)
  - 🔑 对我们的启示: 工具编排层是数字员工系统的核心基础设施问题，单纯扩大工具数量而不改变推理范式将遭遇性能天花板；多工具依赖图的处理能力是下一个差异化维度。

- **Open Agent Leaderboard** by Hugging Face + IBM Research（5月18日）：不评测基础模型，而是评测完整 Agent 系统，揭示同一底层模型在不同 Agent 封装、工具、记忆和错误恢复配置下性能差异巨大——"最强 Agent 系统"和"最强基础模型"不再等价。[链接](https://huggingface.co/blog/ibm-research/open-agent-leaderboard)
  - 🔑 对我们的启示: Agent 工程本身是独立的、可测量的能力维度，这个 Leaderboard 将推动行业将"Agent 封装质量"作为独立评估指标，对 AI-native 产品团队既是压力也是机会。

## 横向洞察 (Cross-cutting Insights)

### 各实验室的发力方向：趋同与分化

本周最清晰的趋同信号是**企业 Agent 基础设施战争的全面白热化**。Anthropic（Self-Hosted Sandboxes + MCP Tunnels + 28项安全合规集成）、Google（Antigravity 2.0 Managed Agents + 93并行子 Agent + WebMCP 标准）、OpenAI（Codex Goal Mode GA + Remote Computer Use + Dell 本地部署）三家实验室本周都在解决同一个问题：让 AI Agent 在企业安全合规约束下可靠运行。竞争的核心已从"谁的模型更聪明"转向"谁的 Agent 运行环境更安全、更可控、更合规"。

分化方面，三家路径截然不同：Google 选择"全栈自建"——从 Gemini 3.5 Flash 模型到 Antigravity IDE 再到 WebMCP 开放 Web 标准，试图成为整个 Agent 生态的基础设施制定者；Anthropic 选择"收购整合"——Stainless SDK + MCP 生态 + 合规 API，通过掌控 SDK 层来控制开发者体验；OpenAI 选择"企业合作部署"——Dell 混合云渠道 + Gartner 领导者定位 + IPO 资本运作，以商业化速度和估值规模建立市场信心。

### 从研究走向产品化的技术趋势

**"自主持续执行"正成为编程 Agent 的核心差异化能力。** Anthropic Claude Code 的 `/goal`、OpenAI Codex 的 Goal Mode GA 和 Remote Computer Use、Google Gemini Spark 的设备关机后云端持续运行——三家都在向同一方向收敛：开发者描述目标，Agent 24/7 自主完成，无需持续在场。LangSmith Engine 的"自主监控→聚类故障→PR 修复"闭环将这一自主性延伸到了生产运维层面。SkillOpt 论文则提供了底层支撑：Agent 可以通过经验优化自身操作手册而无需重新训练——这是"永久在线、持续进化的数字员工"的技术基础。

**安全与可信度从加分项变成入场门票。** Project Glasswing 的公开披露（Mythos 级模型的危险能力）、Anthropic 28项安全集成、OpenAI TanStack 供应链攻击响应、以及 ABA 论文对基准可靠性的系统性质疑，共同揭示了一个趋势：AI Agent 能力越强，安全治理的紧迫性就越高，企业买家对合规证据的要求也越严格。Glasswing 报告的透明性本身就是一种战略选择——主动揭示能力边界，建立负责任 AI 的品牌溢价。

### 文档/课程更新背后的产品战略信号

本周 Anthropic 和 OpenAI 都在通过员工用例和 Academy 课程"教育市场"——Anthropic 用销售/财务团队的 Cowork 用例证明非技术员工可以独立部署 Agent，OpenAI 用 Academy 网络研讨会向财务团队展示 Codex 的业务价值。这不是偶然的内容策略：两家公司都在向企业决策者（而非开发者）传递"ROI 可量化"的信号。Claude Code 将 `/simplify` 更名为 `/code-review` 则是一个更微妙的产品定义信号：Anthropic 正在将代码审查重新定位为 Agent 化开发工作流的原生节点，而非外部审查工具——配合 `--comment` 直接发布 GitHub PR 评论的能力，暗示未来可能深度集成至 CI/CD 管道。

### 对 AI-native Startup 的启示

**本周的核心教训：系统层而非模型层才是护城河。** Hugging Face 的 Open Agent Leaderboard 正式确认了"最强 Agent 系统 ≠ 最强基础模型"——Agent 封装、工具配置、记忆管理和错误恢复能力是独立的、可测量的差异化维度。DeepSeek V4-Pro 永久75%降价之后，纯模型 API 层的利润空间已被极度压缩。对 AI-native 初创公司而言，真正的机会在于：（1）垂直行业深度 Agent Workflow（如 Trend Micro 零日漏洞发现闭环，PwC 保险承保从10周到10天）；（2）自进化 Agent 技能体系（SkillOpt 方向的 `best_skill.md` 生产实践）；（3）Agent 可观测性和自愈基础设施（LangSmith Engine 的"观察→聚类→修复"闭环是值得复制的产品范式）。
