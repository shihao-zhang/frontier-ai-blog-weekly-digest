## 本周重要论文 (Notable Papers)

- **Automated Benchmark Auditing for AI Agents and Large Language Models** (ABA)：提出了一个 Agent 化的基准审计框架，对9个领域168个基准进行系统审计，发现超过25.7%的任务存在关键问题（模糊任务设计、执行环境冲突、错误的 ground truth）。过滤问题任务后，SWE-bench Verified 和 Terminal-Bench 2 上的模型平均性能分别提升9.9%和9.6%，排名发生显著变化。[链接](https://arxiv.org/abs/2605.26079)
  - 🔑 对我们的启示：当前 Agent 基准的可靠性远低于预期，四分之一的任务本身有问题。在评估 Agent 能力时需要更审慎地看待基准分数。

- **Security Risks in Tool-Enabled AI Agents: A Systematic Analysis of Privileged Execution Environments**（已被 IEEE COMPSAC 2026 接收）：系统分析了使用特权工具执行副作用操作的云托管 AI Agent 的安全风险。核心发现：许多 Agent 安全风险是架构层面的，而非利用驱动的——源自过度授权的工具和能力-意图不匹配。[链接](https://arxiv.org/abs/2605.09721)
  - 🔑 对我们的启示：构建 Agent 系统时，工具权限的最小化原则和能力-意图对齐比传统安全防护更重要。

- **Authorization Propagation in Multi-Agent AI Systems: Identity Governance as Infrastructure** by Kamiwaza AI：形式化了多 Agent AI 系统中的"授权传播"问题——在非人类主体跨边界委托任务和综合结果时如何维护授权不变性。识别了三个子问题（传递性委托、聚合推断、时间有效性）和七个结构性需求。[链接](https://arxiv.org/abs/2605.05440)
  - 🔑 对我们的启示：多 Agent 系统的身份治理是一个被严重忽视的基础设施问题，现有的 RBAC/ABAC 模型不足以应对 Agent 时代的授权需求。

- **LLM-Powered AI Agent Systems and Their Applications in Industry** by Guannan Liang, Qianqian Tong：全面综述 LLM 驱动的 Agent 系统如何从规则化前辈演进，分类为软件型、物理型和自适应混合型系统，覆盖客服、软件开发、制造自动化、教育、金融交易和医疗等领域的应用。[链接](https://arxiv.org/abs/2505.16120)
  - 🔑 对我们的启示：Agent 系统正在从通用走向垂直行业深耕，数字员工/AI Worker 产品应聚焦特定行业的 workflow 而非通用能力。

- **Code World Model (CWM) Preparedness Report** by Meta FAIR：Meta 的32B参数开源模型，在 Python 解释器和 Agent Docker 环境的 observation-action 轨迹上训练，经过多任务推理 RL 后在 SWE-bench Verified 上达到65.8%，LiveCodeBench 68.6%，Math-500 96.6%。该报告文档化了安全评估过程。[链接](https://arxiv.org/abs/2605.00932)
  - 🔑 对我们的启示：开源代码生成模型已接近闭源模型的中等水平，post-training RL 是关键差异化手段。

## 横向洞察 (Cross-cutting Insights)

### 实验室发力方向的趋同与分化

本周最显著的趋同信号是**"Agent 基础设施战争"的全面升级**。Anthropic 发布 Self-Hosted Sandboxes 和 MCP Tunnels，Google 推出 Antigravity 2.0 和 Managed Agents API，OpenAI 与 Dell 合作将 Codex 推向本地部署——三大实验室不约而同地在解决同一个问题：让 AI Agent 在企业环境中安全、可控地运行。xAI 的 Grok Skills/Connectors 和 LangChain 的 LangSmith Engine 也在做类似的事情。Agent 竞争的核心已从"谁的模型更聪明"转向"谁的 Agent 基础设施更完善"。

分化方面，Google 选择了"全栈自建"路线（Gemini 模型 + Antigravity IDE + Spark Agent + WebMCP 标准），Anthropic 选择了"收购整合"路线（Stainless SDK + MCP 生态），而 OpenAI 选择了"合作部署"路线（Dell 企业合作 + Deployment Company）。三条路径代表了三种不同的生态构建哲学。

### 从研究走向产品化的技术趋势

**"意图驱动编程"正在成为编程 Agent 的新范式。** OpenAI Codex 的 Appshots（截图驱动）和 Goal Mode（目标驱动），与 Claude Code 的 `/goal`（跨轮目标持续）形成了直接竞争。开发者不再需要告诉 AI "怎么做"，只需要告诉它"做什么"或者"做成什么样"。Gartner 将这一赛道独立为"Enterprise AI Coding Agents"魔力象限的事实本身就是一个强烈信号——Agent 化编程已从探索期进入主流市场。

**安全与治理从"Nice-to-have"变成了"必须有"。** Project Glasswing 揭示了 AI 在攻击面发现上的巨大能力优势，TanStack 供应链攻击暴露了 AI 公司自身的脆弱性，而多篇论文（ABA、Security Risks in Tool-Enabled Agents、Authorization Propagation）都在指向同一个结论：Agent 的能力已经超越了我们的安全治理框架。

### 文档/课程更新背后的产品战略信号

Anthropic 的 Claude Code 更新节奏（`claude agents` 多任务编排 + `/goal` 持续执行 + Plugin 生态）清晰地指向一个战略意图：将 Claude Code 从开发者的"编程助手"重新定位为"自主开发团队"。OpenAI 的 Codex Chrome Extension + 移动端 Codex 则表明其策略是让 Codex 无处不在——浏览器、IDE、手机、企业内网。Google 的 Antigravity 2.0 CLI 硬性替代旧版 Gemini CLI（6月18日截止），反映了其在开发者生态中"破旧立新"的果决态度。

### 对 AI-native Startup 的启示

本周的核心教训是：**Agent 时代的护城河在基础设施层，而非模型层。** 四大实验室的并购潮（Stainless、Contextual AI、Emmi AI、Dreamer）全部瞄准工具链和基础设施，而非模型能力。DeepSeek V4-Pro 永久降价75%则进一步压缩了纯模型层的利润空间。对 AI-native 初创公司而言，最大的机会可能在于：（1）垂直行业的 Agent Workflow（如 PwC 的保险承保案例从10周压缩到10天），（2）Agent 安全治理中间件（授权传播、审计框架），（3）Agent 可观测性和评估工具（LangSmith Engine 的方向）。
